# JMH 介绍与初探

## 背景
### JVM 优化
：JVM 具有许多复杂的特性，如即时编译 (JIT)、垃圾回收 (GC)、逃逸分析 (Escape Analysis) 等。这些特性使得 JVM 在不同情况下的性能表现差异巨大，因此简单的基准测试方法难以准确反映实际性能。

JVM 具有复杂的优化机制，包括即时编译（Just-In-Time Compilation, JIT）、  
垃圾回收（Garbage Collection, GC）、逃逸分析 (Escape Analysis) 等，这些优化可能会影响基准测试的结果。  
这些特性使得 JVM 在不同情况下的性能表现差异巨大，因此简单的基准测试方法难以准确反映实际性能。

### 内联（Inlining）
JVM 在运行时可能会将小的方法内联到调用者中，从而减少方法调用的开销。这可能会导致基准测试结果与实际应用情况不符。

### 死码消除（Dead Code Elimination）
如果基准测试方法的返回值没有被使用，JVM 可能会优化掉整个方法的执行，导致测试结果失真。

### 多线程
在多线程环境下进行基准测试时，需要考虑线程调度和同步开销。

需要一种专门的工具来隔离和控制这些因素，确保测试结果的准确性和稳定性。
